# Loupe

> A native, holographic, domain-agnostic **exploded-view explorer**. Point it at anything with nested
> structure — a neural network, a rocket engine, a protein, a paper, a codebase — and **hold it, explode
> it, and zoom across scales** to inspect any level of detail. The Iron Man "Mark I → Mark II" hologram,
> generalized into a reusable tool.

Loupe is built in **Godot 4.7** with **typed GDScript**, desktop-first, with XR reserved as a first-class
future path. Visuals are a med-high **holographic** style (translucent + emissive rim + depth fog), not
photoreal. Everything is **$0 / offline-first** — no paid services, no API keys for the core.

---

## What it does

A single shell explores wildly different domains because of one separation:

- **The SHELL is universal** — containment hierarchy + semantic zoom/LOD + explode + a device-agnostic
  intent bus (the seam that makes XR "just another input adapter").
- **The LENSES are specialized** — each entity carries a typed *payload*
  (`mesh | matrix | attention | conv | graph | text | equation | …`) dispatched to a renderer that draws
  *that type* beautifully.

> **Universality at the structure level; quality at the payload level.**

### The Model-Architecture arm (flagship)

The most developed arm turns any ML model into an interactive **system diagram** you can dive through:

```
model  →  stage cards (L1)  →  mechanism (L2)  →  heads / sub-parts (L3)
```

- **Focus-dive navigation** — click/`E` to dive into a part; its casing fades and recedes on the Z-axis
  while its internals reveal (engine-cutaway feel). `Q` ascends, `R` recenters.
- **Animated mechanisms** — self-attention computes (Q·Kᵀ → softmax → ·V), convolution slides a padded
  kernel into a feature map, LSTM gates wire a cell-state highway, SSM scans, etc.
- **Real LaTeX** — each component shows its equation, rendered offline to cached PNGs.
- **Right-side panel** — name + bulleted overview/detail/deep description on click.
- **14 models** generated by one adapter: GPT-2, BERT, seq2seq, ViT, AlexNet, ResNet, U-Net, LSTM, GRU,
  Mamba, Stable Diffusion, VAE, GAN, MoE.

## Run it

**Prerequisite:** Godot 4.7 (standard, GDScript build). On Windows it is installed at
`…\WinGet\Packages\GodotEngine.GodotEngine_*\Godot_v4.7-stable_win64.exe`.

```powershell
$GODOT = (Get-ChildItem "C:\Users\$env:USERNAME\AppData\Local\Microsoft\WinGet\Packages\GodotEngine.GodotEngine_*\Godot_v4.7-stable_win64.exe").FullName
cd C:\Code\Self\projects\loupe
& $GODOT --headless --path . --import      # first run only: build the asset cache
& $GODOT --path .                          # open the window
```

## The ingestion package (`ingest/`)

A **standalone Python package** — independent of the engine, which only ever loads IR. It implements the
universal function `visualize(something) → loupe-ir/v1`, deterministic-first and token-frugal: structured
sources compile with **0 tokens**; an LLM is a frontier fallback (off by default).

```powershell
cd ingest
pip install -e .
loupe-ingest "modelarch:gpt2-medium" -o ..\ir\samples\gen\gpt2-medium.loupe.json
python -m pytest -q                        # 38 tests
```

### Add a new model (the whole point of the adapter)

The Model-Architecture arm expands via **one declarative builder** — no engine changes:

1. In `ingest/loupe_ingest/ingestors/model_arch.py`, write a `ModelSpec` builder describing the model's
   stages (and any dive-in internals), then register it in `TEMPLATES`.
2. Add any new equations to `tools/render_equations.py`.
3. Regenerate + verify:
   ```powershell
   python tools\render_equations.py     # LaTeX → assets/eqn/*.png
   python tools\gen_models.py           # adapter → ir/samples/gen/*.loupe.json + catalog.json
   cd ingest; python -m pytest -q       # gate
   ```
The new model appears automatically in the Model-Architecture dropdown.

**Web-first dims:** for transformer-family models the adapter fetches the real HuggingFace `config.json`
(public, no key, cached) so `distilgpt2` shows 6 layers and `gpt2-medium` shows 24 — faithful, never
guessed. Pass `offline=True` (or no network) to fall back to family defaults.

## Verify (headless, CI-friendly)

```powershell
& $GODOT --headless --path . --import
& $GODOT --headless --path . --script res://tools/verify.gd   # → RESULT PASS
cd ingest; python -m pytest -q                                # → 38 passed
```

`verify.gd` loads **and renders** every sample (hand-authored + adapter-generated) through the lens
registry, so a compile/runtime error in any lens fails the gate — not just an IR parse error.

## Design doctrine (the durable rules)

- **Honesty / provenance bar** — real dimensions are labelled; illustrative dynamics are tagged
  "(illustrative)"; dimension-only matrices render as an honest "(schematic)" grid. A viewer can never
  mistake a placeholder for real weights.
- **One adapter per arm** — every model family is data (a template); the engine stays untouched.
- **Intent invariant** — the Stage reacts only to abstract intents (Grab/Rotate/Zoom/Explode/Inspect),
  never raw input. XR is a new adapter, not a rewrite.
- **$0, deterministic, cached** — reproducible, offline-capable, re-runs free.

Full rationale lives in **`decisions.md`** (ADR-style, LP-001 … LP-024).
